jsmn (pronounced like \textquotesingle{}jasmine\textquotesingle{}) is a minimalistic JSON parser in C. It can be easily integrated into resource-\/limited or embedded projects.

You can find more information about JSON format at \href{http://www.json.org/}{\texttt{ json.\+org}}

Library sources are available at \href{https://github.com/zserge/jsmn}{\texttt{ https\+://github.\+com/zserge/jsmn}}

The web page with some information about jsmn can be found at \href{http://zserge.com/jsmn.html}{\texttt{ http\+://zserge.\+com/jsmn.\+html}}\hypertarget{md_src_libs_external_jsmn_1_1_0__r_e_a_d_m_e_autotoc_md0}{}\doxysection{Philosophy}\label{md_src_libs_external_jsmn_1_1_0__r_e_a_d_m_e_autotoc_md0}
Most JSON parsers offer you a bunch of functions to load JSON data, parse it and extract any value by its name. jsmn proves that checking the correctness of every JSON packet or allocating temporary objects to store parsed JSON fields often is an overkill.

JSON format itself is extremely simple, so why should we complicate it?

jsmn is designed to be {\bfseries{robust}} (it should work fine even with erroneous data), {\bfseries{fast}} (it should parse data on the fly), {\bfseries{portable}} (no superfluous dependencies or non-\/standard C extensions). And of course, {\bfseries{simplicity}} is a key feature -\/ simple code style, simple algorithm, simple integration into other projects.\hypertarget{md_src_libs_external_jsmn_1_1_0__r_e_a_d_m_e_autotoc_md1}{}\doxysection{Features}\label{md_src_libs_external_jsmn_1_1_0__r_e_a_d_m_e_autotoc_md1}

\begin{DoxyItemize}
\item compatible with C89
\item no dependencies (even libc!)
\item highly portable (tested on x86/amd64, ARM, AVR)
\item about 200 lines of code
\item extremely small code footprint
\item API contains only 2 functions
\item no dynamic memory allocation
\item incremental single-\/pass parsing
\item library code is covered with unit-\/tests
\end{DoxyItemize}\hypertarget{md_src_libs_external_jsmn_1_1_0__r_e_a_d_m_e_autotoc_md2}{}\doxysection{Design}\label{md_src_libs_external_jsmn_1_1_0__r_e_a_d_m_e_autotoc_md2}
The rudimentary jsmn object is a {\bfseries{token}}. Let\textquotesingle{}s consider a JSON string\+: \begin{DoxyVerb}'{ "name" : "Jack", "age" : 27 }'
\end{DoxyVerb}
 It holds the following tokens\+:


\begin{DoxyItemize}
\item Object\+: {\ttfamily \{ \char`\"{}name\char`\"{} \+: \char`\"{}\+Jack\char`\"{}, \char`\"{}age\char`\"{} \+: 27\}} (the whole object)
\item Strings\+: {\ttfamily \char`\"{}name\char`\"{}}, {\ttfamily \char`\"{}\+Jack\char`\"{}}, {\ttfamily \char`\"{}age\char`\"{}} (keys and some values)
\item Number\+: {\ttfamily 27}
\end{DoxyItemize}

In jsmn, tokens do not hold any data, but point to token boundaries in JSON string instead. In the example above jsmn will create tokens like\+: Object \mbox{[}0..31\mbox{]}, String \mbox{[}3..7\mbox{]}, String \mbox{[}12..16\mbox{]}, String \mbox{[}20..23\mbox{]}, Number \mbox{[}27..29\mbox{]}.

Every jsmn token has a type, which indicates the type of corresponding JSON token. jsmn supports the following token types\+:


\begin{DoxyItemize}
\item Object -\/ a container of key-\/value pairs, e.\+g.\+: {\ttfamily \{ \char`\"{}foo\char`\"{}\+:\char`\"{}bar\char`\"{}, \char`\"{}x\char`\"{}\+:0.\+3 \}}
\item Array -\/ a sequence of values, e.\+g.\+: {\ttfamily \mbox{[} 1, 2, 3 \mbox{]}}
\item String -\/ a quoted sequence of chars, e.\+g.\+: {\ttfamily \char`\"{}foo\char`\"{}}
\item Primitive -\/ a number, a boolean ({\ttfamily true}, {\ttfamily false}) or {\ttfamily null}
\end{DoxyItemize}

Besides start/end positions, jsmn tokens for complex types (like arrays or objects) also contain a number of child items, so you can easily follow object hierarchy.

This approach provides enough information for parsing any JSON data and makes it possible to use zero-\/copy techniques.\hypertarget{md_src_libs_external_jsmn_1_1_0__r_e_a_d_m_e_autotoc_md3}{}\doxysection{Usage}\label{md_src_libs_external_jsmn_1_1_0__r_e_a_d_m_e_autotoc_md3}

\hypertarget{md_src_libs_external_jsmn_1_1_0__r_e_a_d_m_e_autotoc_md4}{}\doxysection{API}\label{md_src_libs_external_jsmn_1_1_0__r_e_a_d_m_e_autotoc_md4}
Token types are described by {\ttfamily jsmntype\+\_\+t}\+: \begin{DoxyVerb}typedef enum {
    JSMN_UNDEFINED = 0,
    JSMN_OBJECT = 1 << 0,
    JSMN_ARRAY = 1 << 1,
    JSMN_STRING = 1 << 2,
    JSMN_PRIMITIVE = 1 << 3
} jsmntype_t;
\end{DoxyVerb}
 {\bfseries{Note\+:}} Unlike JSON data types, primitive tokens are not divided into numbers, booleans and null, because one can easily tell the type using the first character\+:


\begin{DoxyItemize}
\item {\ttfamily \textquotesingle{}t\textquotesingle{}, \textquotesingle{}f\textquotesingle{}} -\/ boolean
\item {\ttfamily \textquotesingle{}n\textquotesingle{}} -\/ null
\item {\ttfamily \textquotesingle{}-\/\textquotesingle{}, \textquotesingle{}0\textquotesingle{}..\textquotesingle{}9\textquotesingle{}} -\/ number
\end{DoxyItemize}

Token is an object of {\ttfamily jsmntok\+\_\+t} type\+: \begin{DoxyVerb}typedef struct {
    jsmntype_t type; // Token type
    int start;       // Token start position
    int end;         // Token end position
    int size;        // Number of child (nested) tokens
} jsmntok_t;
\end{DoxyVerb}
 {\bfseries{Note\+:}} string tokens point to the first character after the opening quote and the previous symbol before final quote. This was made to simplify string extraction from JSON data.

All job is done by {\ttfamily \mbox{\hyperlink{structjsmn__parser}{jsmn\+\_\+parser}}} object. You can initialize a new parser using\+: \begin{DoxyVerb}jsmn_parser parser;
jsmntok_t tokens[10];

jsmn_init(&parser);

// js - pointer to JSON string
// tokens - an array of tokens available
// 10 - number of tokens available
jsmn_parse(&parser, js, strlen(js), tokens, 10);
\end{DoxyVerb}
 This will create a parser, and then it tries to parse up to 10 JSON tokens from the {\ttfamily js} string.

A non-\/negative return value of {\ttfamily jsmn\+\_\+parse} is the number of tokens actually used by the parser. Passing NULL instead of the tokens array would not store parsing results, but instead the function will return the number of tokens needed to parse the given string. This can be useful if you don\textquotesingle{}t know yet how many tokens to allocate.

If something goes wrong, you will get an error. Error will be one of these\+:


\begin{DoxyItemize}
\item {\ttfamily JSMN\+\_\+\+ERROR\+\_\+\+INVAL} -\/ bad token, JSON string is corrupted
\item {\ttfamily JSMN\+\_\+\+ERROR\+\_\+\+NOMEM} -\/ not enough tokens, JSON string is too large
\item {\ttfamily JSMN\+\_\+\+ERROR\+\_\+\+PART} -\/ JSON string is too short, expecting more JSON data
\end{DoxyItemize}

If you get {\ttfamily JSMN\+\_\+\+ERROR\+\_\+\+NOMEM}, you can re-\/allocate more tokens and call {\ttfamily jsmn\+\_\+parse} once more. If you read json data from the stream, you can periodically call {\ttfamily jsmn\+\_\+parse} and check if return value is {\ttfamily JSMN\+\_\+\+ERROR\+\_\+\+PART}.
