JSON-\/C implements a reference counting object model that allows you to easily construct JSON objects in C, output them as JSON formatted strings and parse JSON formatted strings back into the C representation of JSON objects. It aims to conform to \href{https://tools.ietf.org/html/rfc7159}{\texttt{ RFC 7159}}.

\hypertarget{index_autotoc_md16}{}\doxysubsubsection{Building with partial threading support}\label{index_autotoc_md16}
Although json-\/c does not support fully multi-\/threaded access to object trees, it has some code to help make its use in threaded programs a bit safer. Currently, this is limited to using atomic operations for \mbox{\hyperlink{json__object_8h_a675aa3a9cced685dbfd1c1a770a0c3e4}{json\+\_\+object\+\_\+get()}} and \mbox{\hyperlink{json__object_8h_afabf61f932cd64a4122ca8092452eed5}{json\+\_\+object\+\_\+put()}}.

Since this may have a performance impact, of at least 3x slower according to \href{https://stackoverflow.com/a/11609063}{\texttt{ https\+://stackoverflow.\+com/a/11609063}}, it is disabled by default. You may turn it on by adjusting your cmake command with\+: -\/DENABLE\+\_\+\+THREADING=ON

Separately, the default hash function used for object field keys, lh\+\_\+char\+\_\+hash, uses a compare-\/and-\/swap operation to ensure the random seed is only generated once. Because this is a one-\/time operation, it is always compiled in when the compare-\/and-\/swap operation is available.\hypertarget{index_autotoc_md17}{}\doxysubsubsection{cmake-\/configure wrapper script}\label{index_autotoc_md17}

\hypertarget{index_autotoc_md22}{}\doxysubsection{Using json-\/c $<$a name=\char`\"{}using\char`\"{}$>$}\label{index_autotoc_md22}
To use json-\/c you can either include \mbox{\hyperlink{json_8h}{json.\+h}}, or preferably, one of the following more specific header files\+:


\begin{DoxyItemize}
\item \mbox{\hyperlink{json__object_8h}{json\+\_\+object.\+h}} -\/ Core types and methods.
\item \mbox{\hyperlink{json__tokener_8h}{json\+\_\+tokener.\+h}} -\/ Methods for parsing and serializing json-\/c object trees.
\item \mbox{\hyperlink{json__pointer_8h}{json\+\_\+pointer.\+h}} -\/ JSON Pointer (RFC 6901) implementation for retrieving objects from a json-\/c object tree.
\item \mbox{\hyperlink{json__object__iterator_8h}{json\+\_\+object\+\_\+iterator.\+h}} -\/ Methods for iterating over single json\+\_\+object instances. (See also {\ttfamily \mbox{\hyperlink{json__object_8h_acf5f514a9e0061c10fc08055762639ee}{json\+\_\+object\+\_\+object\+\_\+foreach()}}} in \mbox{\hyperlink{json__object_8h}{json\+\_\+object.\+h}})
\item \mbox{\hyperlink{json__visit_8h}{json\+\_\+visit.\+h}} -\/ Methods for walking a tree of json-\/c objects.
\item \mbox{\hyperlink{json__util_8h}{json\+\_\+util.\+h}} -\/ Miscellaneous utility functions.
\end{DoxyItemize}

For a full list of headers see \href{https://json-c.github.io/json-c/json-c-current-release/doc/html/files.html}{\texttt{ files.\+html}}

The primary type in json-\/c is json\+\_\+object. It describes a reference counted tree of json objects which are created by either parsing text with a \mbox{\hyperlink{structjson__tokener}{json\+\_\+tokener}} (i.\+e. {\ttfamily \mbox{\hyperlink{json__tokener_8h_a61679f178111963a9ffa3c8179553f7a}{json\+\_\+tokener\+\_\+parse\+\_\+ex()}}}), or by creating (with {\ttfamily \mbox{\hyperlink{json__object_8h_a68c383f54544fca19b5f2425be397600}{json\+\_\+object\+\_\+new\+\_\+object()}}}, {\ttfamily \mbox{\hyperlink{json__object_8h_ae92f0770fb4b3c884ce35de52d3d7de8}{json\+\_\+object\+\_\+new\+\_\+int()}}}, etc...) and adding (with {\ttfamily \mbox{\hyperlink{json__object_8h_a27bd808a022251059a43f1f6370441cd}{json\+\_\+object\+\_\+object\+\_\+add()}}}, {\ttfamily \mbox{\hyperlink{json__object_8h_a18cdd9a7455e09f36cdf6e5756b7f586}{json\+\_\+object\+\_\+array\+\_\+add()}}}, etc...) them individually. Typically, every object in the tree will have one reference, from its parent. When you are done with the tree of objects, you call \mbox{\hyperlink{json__object_8h_afabf61f932cd64a4122ca8092452eed5}{json\+\_\+object\+\_\+put()}} on just the root object to free it, which recurses down through any child objects calling \mbox{\hyperlink{json__object_8h_afabf61f932cd64a4122ca8092452eed5}{json\+\_\+object\+\_\+put()}} on each one of those in turn.

You can get a reference to a single child ({\ttfamily \mbox{\hyperlink{json__object_8h_a1a097805abb53b4c8a60d573730a8939}{json\+\_\+object\+\_\+object\+\_\+get()}}} or {\ttfamily \mbox{\hyperlink{json__object_8h_a676711a76545d4ec65cc75f100f5fd19}{json\+\_\+object\+\_\+array\+\_\+get\+\_\+idx()}}}) and use that object as long as its parent is valid. ~\newline
 If you need a child object to live longer than its parent, you can increment the child\textquotesingle{}s refcount ({\ttfamily \mbox{\hyperlink{json__object_8h_a675aa3a9cced685dbfd1c1a770a0c3e4}{json\+\_\+object\+\_\+get()}}}) to allow it to survive the parent being freed or it being removed from its parent ({\ttfamily \mbox{\hyperlink{json__object_8h_ac6605fdafca20bd5d33c84f4f80a3bda}{json\+\_\+object\+\_\+object\+\_\+del()}}} or {\ttfamily \mbox{\hyperlink{json__object_8h_a722eca9f578704d3af38b97549242c1f}{json\+\_\+object\+\_\+array\+\_\+del\+\_\+idx()}}})

When parsing text, the \mbox{\hyperlink{structjson__tokener}{json\+\_\+tokener}} object is independent from the json\+\_\+object that it returns. It can be allocated ({\ttfamily \mbox{\hyperlink{json__tokener_8h_a5ac7e2c350bc592cf2fa7b9935b00ef5}{json\+\_\+tokener\+\_\+new()}}}) used one or multiple times ({\ttfamily \mbox{\hyperlink{json__tokener_8h_a61679f178111963a9ffa3c8179553f7a}{json\+\_\+tokener\+\_\+parse\+\_\+ex()}}}, and freed ({\ttfamily \mbox{\hyperlink{json__tokener_8h_a887c4661906fc6b36cc366304e522534}{json\+\_\+tokener\+\_\+free()}}}) while the json\+\_\+object objects live on.

A json\+\_\+object tree can be serialized back into a string with {\ttfamily \mbox{\hyperlink{json__object_8h_a9db613127bd4ef7db42307e43a85fc1b}{json\+\_\+object\+\_\+to\+\_\+json\+\_\+string\+\_\+ext()}}}. The string that is returned is only valid until the next \char`\"{}to\+\_\+json\+\_\+string\char`\"{} call on that same object. Also, it is freed when the json\+\_\+object is freed.
